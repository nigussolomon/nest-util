What you can launch with Nest-Util

Everything your backend team needs to move from idea to production.

@nest-util/nest-crud

Generate and scale CRUD operations with reusable controllers, filters, response interceptors, and pagination utilities.

  • Controller + service scaffolding consistency
  • Type-safe DTO workflows
  • Cleaner business logic, less boilerplate

@nest-util/nest-auth

Add complete authentication flows with JWT support, refresh token handling, and modular route protection.

  • Customizable auth options
  • Built-in route guards and decorators
  • Improved security posture by default

@nest-util/nest-file

Manage secure uploads with encrypted object storage, owner attachments, and metadata persistence.

  • Storage abstraction for production systems
  • Encryption for sensitive files
  • Ownership and access-friendly structure

NCNU CLI Generator

Generate entities, DTOs, services, and controllers fast using conventions your whole team can rely on.

  • CLI-powered productivity boost
  • Consistent project architecture
  • Less repetitive coding work

Frequently asked questions

Questions teams usually ask before adopting Nest-Util.

Is Nest-Util only for large teams?

No. Solo developers and startups benefit immediately from faster scaffolding and stronger architecture defaults.

Can I adopt modules one by one?

Yes. Each package is modular, so you can integrate only what your project needs and expand over time.

Does it work with existing NestJS projects?

Absolutely. Nest-Util is designed to integrate into existing apps with minimal friction and clear boundaries.
